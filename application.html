Currently, some students who study IT still confuse the concept of website and web app, have a friend to ask, so I write this article by the way.

This is a question "easy to think but not easy", because the boundary between website and webapp is quite fragile. I have to synthesize a lot of answers from stackoverflow and programmers.stackexchange to give me the most "approximate" answer.

Web Site and Web Appp definition

WEBSITE: A long time ago,when the Internet was rudimentary, the web was written in single html. Every single website is written in html called Web Page. Gather multiple individual websites, into one large website, with the same domain name, called Website.

WEBAPP: Understandably, web apps are web-based applications. Through the web app, users can perform a number of tasks: calculating, sharing images, shopping ... The interactive nature of the web app is much higher than the website.

For some people who are not knowledgeable about IT, all things online, accessible by browser are websites. So they often ask you: supermarket management website, sales website, ... they are all webapps.

Different Between Web App And Web Site
Web Site Web App
Low interaction, low functionality (See, read, click back and forth between links ...) High interaction, multiple functions (Post information, upload files, export reports ...)
Made up of static html pages and some resources (images, audio, video) Created by html and back end code (PHP, C #, Java, ...)
Used to store and display information Used to "perform a task", perform the functions of an application

Compare website and web app

In fact, the boundary between web app and website is quite fragile. An online newspaper - vnexpress, for example, in the eyes of readers it is a website.
